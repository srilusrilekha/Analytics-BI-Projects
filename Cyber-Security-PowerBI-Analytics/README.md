# Enterprise Cyber Security Power BI Analytics

## Project Overview

This project delivers a centralized Power BI reporting solution designed to provide executive and operational visibility into enterprise cybersecurity metrics across governance, risk, compliance, vulnerability management, and security operations.

The solution transforms curated datasets from an AWS-based cloud data platform into interactive dashboards, enabling stakeholders to monitor cyber posture, track KPIs, analyze trends, and support data-driven decision-making through a standardized reporting experience.

## Business Problem

Enterprise cybersecurity metrics were distributed across multiple operational systems, making it difficult to provide consistent and executive-level reporting.

Key business challenges included:

- Fragmented cybersecurity reporting
- Multiple disconnected reporting sources
- Lack of standardized KPI definitions
- Manual reporting efforts
- Limited executive visibility
- Inconsistent business logic across reports

A centralized reporting solution was required to provide consistent, interactive, and business-friendly analytics.

## Data Modeling Approach

The reporting solution utilizes a **Star Schema** semantic model following Microsoft Power BI best practices.

### Why Star Schema?

The solution supports enterprise cybersecurity reporting across multiple business domains while maintaining high performance and simplified report development.

The Star Schema was implemented to:

- Simplify relationships between reporting tables
- Improve dashboard performance
- Enable reusable DAX measures
- Centralize business logic
- Support scalable reporting across multiple dashboards
- Improve model maintainability

The semantic model includes:

- Central Fact Table (VM_Metrics)
- Dimension Tables
- Dedicated Measures Table
- Supporting Lookup Tables

This architecture provides a scalable and reusable reporting foundation while maintaining consistent business logic across the solution.

## Data Refresh & Monitoring

The reporting solution supports scheduled enterprise reporting by refreshing curated cybersecurity datasets from the centralized semantic model on a daily basis.

### Refresh Schedule

The Power BI dataset is refreshed **once daily** to provide stakeholders with an up-to-date view of enterprise cybersecurity metrics.

**Scheduled Refresh Window**

- **07:00 AM EST (Daily)**

This refresh schedule aligns with executive and governance reporting requirements, ensuring that business users begin each day with the latest validated cybersecurity metrics and KPIs.

### Monitoring & Optimization

To ensure reporting reliability, accuracy, and performance:

- Dataset refreshes are continuously monitored for successful execution.
- Refresh failures and data discrepancies are investigated and resolved promptly.
- Report performance is periodically reviewed and optimized.
- DAX measures and semantic model performance are continuously tuned.
- Dashboard enhancements are implemented based on evolving business requirements and stakeholder feedback.
