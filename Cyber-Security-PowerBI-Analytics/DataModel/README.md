# DataModel

## Overview

This folder contains the architecture artifacts for the **Enterprise Cyber Security Power BI Analytics** solution.

Unlike the companion Data Engineering repository, which focuses on cloud architecture and ETL pipelines, this folder documents the Business Intelligence layer responsible for transforming curated datasets into interactive dashboards and executive reporting.

The architecture follows Power BI best practices by implementing a reusable semantic model built on a Star Schema design, enabling scalable reporting, simplified maintenance, and high-performance analytics.

# 2. Power BI Semantic Model (Star Schema)

**File**

`2_PowerBI_SemanticModel_Star_Schema.png`

### Purpose

Represents the logical semantic model used by the Power BI solution.

The model follows a **Star Schema** design to improve query performance, simplify report development, and provide a scalable foundation for enterprise reporting.

### Modeling Approach

The semantic model consists of:

- Central Fact Table
- Dimension Tables
- Dedicated Measures Table
- Supporting Lookup Tables

This structure minimizes complex relationships while maximizing report performance and DAX efficiency.

---

## Data Model Components

### Fact Table

**VM_Metrics**

Stores the transactional and aggregated business metrics consumed by Power BI visuals.

---

### Dimension Tables

The reporting model uses dimension tables to provide business context, filtering, and categorization.

Examples include:

- VM_Master_Metrics
- Time Period
- Metric Name Lookup

---

### Measures Table

A dedicated **Measures Table** is implemented to centralize all DAX measures used throughout the reporting solution.

This approach improves:

- Model organization
- Measure discoverability
- Reusability
- Long-term maintainability

---

## Modeling Best Practices

The semantic model was designed following Microsoft Power BI best practices.

### Key Design Principles

- Star Schema implementation
- Single-direction relationships
- Dedicated DAX Measures Table
- Optimized filtering paths
- Simplified relationship model
- Reusable semantic layer
- Performance-optimized reporting model
